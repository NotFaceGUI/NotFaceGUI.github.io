//定义一个无参方法
public static void 方法名称(){
	//方法体
}

修饰符 返回值类型 方法名称(参数类型 参数名称，………… ) {
	方法体；
	return 返回值；
}
例如:
/*
修饰符：public static
返回值类型：int
方法名称：sum
*/

5.4、方法重载（OverLoad）

  • 方法名称一样，但参数列表不同
  • 只需要一个方法名，就可以实现多个类似的功能
  • 重载因素
    • 参数个数不同
    • 参数类型不同
    • 参数的多类型顺序不同
  • 重载因素无关
    • 与参数名称无关
    • 与返回类型无关

6.3.1、For循环

for (初始化表达式①;布尔表达式②;步进表达式④) {
    循环体③;
}
↓ //如下！

    System.out.println(i);//打印0~9的数字
}

执行过程：

  • ①②③④→②③④...直到②不满足时跳出循环

6.3.2、while循环（当循环）前测

初始化表达式;
while (条件表达式) {// 直到不满足条件退出循环
    循环体;
    步进表达式;
}

6.3.3、do...while循环（直到循环）后测

初始化语句;
do{
    循环体;
    步进表达式;
}while(条件表达式); //当 条件不成立 时退出循环

6.3.4、三种循环的区别

  • 当初始化语句一开始就不满足条件表达式，那么For和while执行0次，do-while执行一次
  • For循环的变量**i**只有在for循环内有用，超出for循环范围后没用（生命周期|局部变量）

九、面向对象

**面向过程：**将一个项目（或者一个事件）从头到尾按顺序，一步一步完成，先做什么，后做什么，一直到结束，也是我们人做事的方法。

**面向对象：**将一个项目（或者一个事件）分成更小的项目，每一个部分负责一方面的功能，最后由这些部分组成一个整体。

面向过程（Procedure Oriented）是分析解决问题的步骤，然后用函数把这些步骤一步一步的实现，然后在使用的时候一一调用则可。强调的是完成这件事儿的动作，更接近我们日常处理事情的思维。

面向对象（Object Oriented）把构成问题的transaction分解成各个对象，而建立对象的目的也不是为了完成一个个步骤，而是为了描述某个事物在解决整个问题的过程中所发生的行为，意在写出通用代码，加强代码重用，屏蔽差异性。

9.4.3、多态性

类与类之间的继承(extends)，或者接口与实现类的实现(implement)，是多态性的前提。

必须有上下级关系，才能有多态性的出现！

**一个对象拥有多个形态，这就是 对象的多态性 **

譬如：

定义一个 人类 在定义一个 学生类 使其继承于 人类，那么实例化学生类的对象，这个对象既是学生又是人

格式：

// 父类引用指向子类对象
父类名称 对象名 = new 子类名称()

接口名称 对象名 = new 实现类名称()

成员变量的访问特点：

等号左边是谁就优先用谁，没有则向上查找

例如：

父类中有个

int num = 10;

子类继承于父类同时也定义一个：

int num = 20;

如果实例化子类的对象

Zi obj = new obj();
obj.num;

调用的则是等号左边子类的num变量

但如果父类引用指向子类对象：

Fu obj = new Zi();
obj.num;

则调用的是Fu类中的num变量

值得一提的是，成员变量无法被子类或者实现类覆写，方法可以被覆写，所以当用父类引用指向子类对象时，想要调用子类变量可通过，间接方法调用，通过子类覆写以实现调用之类成员变量的目的。

方法

父子都有方法则new的是谁调用的就是谁的方法

Fu obj = new Zi();

编译看左边，运行看右边

接口使用步骤：

// 接口需要有一个实现类来实现该接口

注意：接口的实现类必须覆写（实现）接口中所有的 抽象方法。如果没有全部覆写接口中的方法那么除非你是 抽象类不然就会报错！

  1. 接口是没有静态代码块和构造方法的。
  2. 一个类可以实现多个接口
  3. 如果实现类所实现的多个接口当中，出现重复的抽象方法，只需要覆写一个即可。
  4. 如果实现类没有覆写接口中的所有抽象方法，那么实现类必须是抽象类。
  5. 如果实现类所实现的接口中，存在多个默认方法，那么实现类要对冲突的默认方法进行覆盖重写。
  6. 如果一个类当中的父类中的一个方法，和接口当中的默认方法，产生了冲突，那么优先使用父类方法。

  3. 接口和接口之间可以多继承

匿名内部类

如果接口的实现类（父类的子类），只需要使用唯一的一次。那么就可以省略掉改类的定义，改为使用 匿名内部类。

定义格式

接口名称 对象名 = new 接口名称(){
    // 覆盖重写接口中所有的抽象方法
};

类名称 对象名 = new 类名称(){
    // 覆盖重写抽象类中所有的抽象方法
};

new 接口名称() {...}

解析：

  1. new 代表创建对象的动作
  2. 接口名称就是匿名内部类需要实现的接口
  3. {...} 才是匿名内部类的内容

注意：

  1. 匿名内部类，在创建对象时，只能使用唯一一次，多次使用得用实现类
